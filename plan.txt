1. EXECUTIVE SUMMARY & PROJECT VISION
1.1 The Core Problem: Context Blindness
Modern mobile microlearning platforms offer bite-sized, short-duration units designed to reduce cognitive load and support learning on the go. However, empirical evaluation of corporate and open micro-course frameworks (such as Duolingo, Axonify, and LinkedIn Learning) highlights a fundamental architectural flaw: Context Blindness.
Traditional applications are entirely reactive and static. They assume that the user is always in an optimal learning environment—stationary, focused, and equipped with a stable, high-bandwidth internet connection. When a system delivers a heavy, data-intensive video lesson to a student who is navigating a dead-zone on a cellular network, or when it requires precise text input while a user is actively walking, it introduces severe extraneous cognitive overload. This friction leads directly to user frustration and high drop-out rates.
1.2 The Proposed Solution
This project details the engineering specification for a proactive, Context-Aware Adaptive Microlearning Application. By bypassing intermediate server layers and connecting a native Flutter client directly to a cloud-hosted Supabase PostgreSQL backend, the application shifts digital education from reactive delivery to real-time ambient adaptation.
The user experience centers around an infinite, vertically snapping, TikTok-style swiping feed built using Flutter's native layout engine. Instead of forcing users to manually browse or search for content, the application continuously reads low-level hardware telemetry:
* Kinetic Motion Sensing: Uses the device's physical accelerometer array to track environmental movement vectors. If continuous high-variance movement is detected (e.g., walking), the UI dynamically swaps complex lessons out for high-target, large-button True/False interactive quizzes to prevent mis-clicks and minimize cognitive strain while in motion.
* Network Path Awareness: Monitored via cellular sub-system channels. If the available bandwidth degrades (e.g., weak cellular edge state), the client engine intercepts the active content pipeline, drops heavy video asset requests, and seamlessly serves an elegant, pre-cached text lesson with high-contrast scalable typography.
2. SYSTEM ARCHITECTURE & EVOLUTIONARY PROGRESSION
2.1 Technical Architectural Re-Alignment
The project's implementation strategy evolved significantly through iterative optimization cycles:






[Phase 1 Baseline Concept]
Flutter Client ---> Node.js / NestJS API Middleware (Prisma ORM) ---> Supabase Cloud DB

[Optimized Production Architecture]
Flutter Client (Riverpod State + PostgREST) <=========================> Supabase Cloud DB (PostgreSQL)

Initially, the project was planned around a standard multi-tiered layout using a Node.js/NestJS application middleware server mapped via Prisma ORM to a Supabase database. While this architectural pattern is highly effective for legacy web systems, it introduces unnecessary latency overhead, extra server-hosting costs, and redundant object-relational mapping steps within a client-side streaming environment.
By eliminating Prisma and the middle-tier API server entirely, the production architecture utilizes direct secure client queries via Supabase’s native PostgREST high-performance database wrapper. The Flutter application handles state propagation natively, converting the phone into a client-side context-resolution engine. This design ensures rapid data processing and highly responsive UI transitions during rapid feed swipes.
2.2 System Framework Selection & Packaging
To establish direct communication with both cloud infrastructure channels and core operating system hardware components, the client runtime environment utilizes the following production-grade dependencies:
* supabase_flutter: Connects directly to the PostgreSQL database layer, handles secure user authentication sessions, and handles transactional logging commands.
* connectivity_plus: Connects directly to system network monitors to instantly report changes in connectivity states (Wi-Fi, Cellular, or Dead-Zone profiles).
* sensors_plus: Opens high-frequency hardware channels to capture real-time raw linear data streams from the phone's physical accelerometer chip.
* flutter_riverpod: Establishes an immutable, globally accessible client-side memory storage layer that manages the lifecycle of data streams and prevents memory leakage.

3.3 Phase 3: Building the Ambient Hardware Sensor-Sensing Engine
The sensing engine runs continuously in the application's background layer, transforming messy hardware signals into clean, logic-driven application states.
3.3.1 Mathematical Motion Detection Model
To detect motion without draining battery life, the engine monitors the device's physical acceleration vector over a rolling sequence:


$$\text{Vector Magnitude } (A_m) = \sqrt{x^2 + y^2 + z^2}$$
The system continuously tracks this magnitude within a moving 3-to-5 second window. When the rolling statistical variance ($\sigma^2$) exceeds a preset threshold ($\tau = 1.8 \text{ m/s}^2$), the system flags the user's state as isInMotion = true.
3.3.2 Implementation of the Multi-Stream Core Engine
Create the core tracking service at lib/features/context_engine/services/context_engine_service.dart:






Dart
import 'dart:async';
import 'dart:math';
import 'sensors_plus.dart'; // Abstracted proxy target for tracking architecture
import 'connectivity_plus.dart';

enum AppNetworkStrength { weak, medium, strong }

class UserContextState {
 final bool isInMotion;
 final AppNetworkStrength networkStrength;

 UserContextState({required this.isInMotion, required this.networkStrength});
}

class ContextEngineService {
 final StreamController<UserContextState> _contextStreamController = 
     StreamController<UserContextState>.broadcast();

 Stream<UserContextState> get contextStream => _contextStreamController.stream;

 // Active tracking state properties
 bool _currentMotionState = false;
 AppNetworkStrength _currentNetworkStrength = AppNetworkStrength.strong;

 StreamSubscription? _accelerometerSub;
 StreamSubscription? _connectivitySub;

 ContextEngineService() {
   _initializeSensingPipelines();
 }

 void _initializeSensingPipelines() {
   // 1. Network Sub-System Stream Interception
   _connectivitySub = Connectivity().onConnectivityChanged.listen((connectivityResults) {
     // Access primary connection indicator from multi-interface sequence arrays
     final primaryResult = connectivityResults.first;
     if (primaryResult == ConnectivityResult.wifi) {
       _currentNetworkStrength = AppNetworkStrength.strong;
     } else if (primaryResult == ConnectivityResult.mobile) {
       _currentNetworkStrength = AppNetworkStrength.medium;
     } else {
       _currentNetworkStrength = AppNetworkStrength.weak;
     }
     _emitCombinedState();
   });

   // 2. Linear Kinetic Vector Variational Sequence Sampler
   List<double> accelerationWindow = [];
   
   _accelerometerSub = accelerometerEvents.listen((AccelerometerEvent event) {
     double magnitude = sqrt(event.x * event.x + event.y * event.y + event.z * event.z);
     accelerationWindow.add(magnitude);

     // Maintain rolling execution footprint constraints
     if (accelerationWindow.length > 25) {
       accelerationWindow.removeAt(0);
     }

     if (accelerationWindow.length >= 10) {
       double mean = accelerationWindow.reduce((a, b) => a + b) / accelerationWindow.length;
       double variance = accelerationWindow.map((x) => pow(x - mean, 2)).reduce((a, b) => a + b) / accelerationWindow.length;

       bool structuralMovingTarget = variance > 1.8;
       if (structuralMovingTarget != _currentMotionState) {
         _currentMotionState = structuralMovingTarget;
         _emitCombinedState();
       }
     }
   });
 }

 void _emitCombinedState() {
   if (!_contextStreamController.isClosed) {
     _contextStreamController.add(
       UserContextState(
         isInMotion: _currentMotionState,
         networkStrength: _currentNetworkStrength,
       ),
     );
   }
 }

 void dispose() {
   _accelerometerSub?.cancel();
   _connectivitySub?.cancel();
   _contextStreamController.close();
 }
}

import 'package:supabase_flutter/supabase_flutter.dart';
import '../../context_engine/services/context_engine_service.dart';
import '../models/lesson_model.dart';

class LearningRepository {
 final SupabaseClient _supabase;

 LearningRepository(this._supabase);

 Future<List<LessonModel>> fetchAdaptiveViewportFeed({
   required String userUuid,
   required UserContextState ambientContext,
 }) async {
   // 1. Fetch historical data to filter out completed modules
   final List<dynamic> historyData = await _supabase
       .from('user_progress')
       .select('lesson_id')
       .eq('user_id', userUuid);

   List<int> completedLessonIds = historyData
       .map((record) => record['lesson_id'] as int)
       .toList();

   // 2. Initialize the PostgREST statement builder
   var query = _supabase.from('lessons').select();

   // Exclusion filter: drop completed content IDs to prevent repetition
   if (completedLessonIds.isNotEmpty) {
     query = query.not('id', 'in', completedLessonIds);
   }

   // 3. Dynamic Hardware Filter Evaluation
   if (ambientContext.isInMotion) {
     // Restrict options to motion-safe elements (audio tracks or quizzes)
     query = query.eq('safe_for_motion', true);
   }

   switch (ambientContext.networkStrength) {
     case AppNetworkStrength.weak:
       // Exclude videos and heavy media formats
       query = query.in_('min_network_strength', ['weak']).not('format', 'eq', 'video');
       break;
     case AppNetworkStrength.medium:
       query = query.in_('min_network_strength', ['weak', 'medium']);
       break;
     case AppNetworkStrength.strong:
       // Open the full content catalog across all formats
       break;
   }

   final response = await query.order('id', ascending: true).limit(5);
   return (response as List).map((row) => LessonModel.fromJson(row)).toList();
 }

 Future<void> logLessonCompletionPercentage({
   required String userUuid,
   required int lessonId,
   required int score,
 }) async {
   await _supabase.from('user_progress').upsert({
     'user_id': userUuid,
     'lesson_id': lessonId,
     'score': score,
     'completed_at': DateTime.now().toIso8601String(),
   });
 }
}

class MainSwipeFeedScreen extends StatefulWidget {

 @override
 State<MainSwipeFeedScreen> createState() => _MainSwipeFeedScreenState();

class _MainSwipeFeedScreenState extends State<MainSwipeFeedScreen> {
 final PageController _pageController = PageController();

 @override
 Widget build(BuildContext context) {
   return Scaffold(
     body: Consumer(
       builder: (context, ref, child) {
         final engine = ref.watch(contextServiceProvider);

         return StreamBuilder<UserContextState>(
           stream: engine.contextStream,
           builder: (context, contextSnapshot) {
             if (contextSnapshot.hasError) {
               return Center(child: Text("Hardware Interface Error: ${contextSnapshot.error}"));
             }
             if (!contextSnapshot.hasData) {
               return const Center(child: CircularProgressIndicator());
             }

             final currentContext = contextSnapshot.data!;

               loading: () => const Center(child: CircularProgressIndicator()),
               error: (err, stack) => Center(child: Text("Data Pipeline Fault: $err")),
               data: (lessonsList) {
                 if (lessonsList.isEmpty) {
                   return const Center(child: Text("No contextually valid lessons found. Check settings."));
                 }

                 return PageView.builder(
                   controller: _pageController,
                   scrollDirection: Axis.vertical, // Implements the TikTok-style infinite snapping effect
                   itemCount: lessonsList.length,
                   itemBuilder: (context, index) {
                     final item = lessonsList[index];
                     return _buildAdaptiveLayoutRouter(item);
                   },
                 );
               },
             );
           },
         );
       },
     ),
   );
